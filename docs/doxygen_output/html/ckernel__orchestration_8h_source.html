186 // Decode-style layer forward for autoregressive generation.
187 //
188 // Computes only a single token at `token_index`, while attending over the
189 // KV-cache stored in `p->k`/`p->v` in head-major cache layout:
190 // k/v: [num_kv_heads, cache_capacity, aligned_head_dim]
191 //
192 // The caller is responsible for:
193 // - ensuring `p->k`/`p->v` already contain tokens [0..token_index-1]
194 // - setting `p->rope_pos_offset` to the absolute position for this token
195 // - passing a matching `cache_capacity` (usually model context_window)
