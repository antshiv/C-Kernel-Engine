230 // Decode-style layer forward for autoregressive generation.
231 //
232 // Computes only a single token at `token_index`, while attending over the
233 // KV-cache stored in `p->k`/`p->v` in head-major cache layout:
234 // k/v: [num_kv_heads, cache_capacity, aligned_head_dim]
235 //
236 // The caller is responsible for:
237 // - ensuring `p->k`/`p->v` already contain tokens [0..token_index-1]
238 // - setting `p->rope_pos_offset` to the absolute position for this token
239 // - passing a matching `cache_capacity` (usually model context_window)

241  int token_index,

260 /* ============================================================================
261  * Quantized (Q4_K / Q4_K_M) inference orchestration
262  *
263  * These entry points mirror the fp32 paths but accept weight matrices stored
264  * in GGML-compatible Q4_K blocks. Activations remain fp32 by default; the
265  * decode path can switch to Q8_K activations via CK_Q8K_ACTIVATIONS=1.
266  *
267  * Design note:
268  * - If you enable Q4_K weights, ensure the relevant K dimensions are a
269  * multiple of 256 (QK_K). The engine keeps the quantized weights in their
270  * compact block form and dequantizes on-the-fly inside GEMM/GEMV kernels.
271  * ============================================================================ */
